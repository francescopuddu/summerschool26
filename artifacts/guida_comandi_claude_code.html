01 Git & GitHub in 5 minuti

Prima dei comandi: versionare il codice

Quando lavori con Claude Code modifichi dei file. Git è lo strumento che tiene la storia di quelle modifiche — così puoi tornare indietro in qualsiasi momento e non perdere mai il lavoro.

Versionare

Significa salvare delle istantanee del progetto man mano che lavori. Ogni istantanea è una foto di com'erano i file in quel momento.

Se qualcosa si rompe, torni all'istantanea di prima. È il "salva partita" del codice.

GitHub

È il posto nel cloud dove queste istantanee vengono conservate. Una copia online del tuo progetto.

Serve come backup e per condividere il lavoro con altri (o con i tuoi altri computer).

Locale vs cloud. Git vive sul tuo computer (locale). GitHub è il servizio online. Git funziona anche senza GitHub — GitHub serve per mettere il codice al sicuro e condividerlo.

Creare una repository

Una repo (repository) è semplicemente la cartella di un progetto messa sotto controllo di Git. Si parte così:

Avvio di un progetto

git init  → dice a Git di iniziare a tracciare questa cartella.

Poi si crea la repo gemella su GitHub (dal sito, pulsante New repository) e si collega. Da quel momento le tue istantanee possono salire online.

In pratica Claude Code può eseguire questi comandi per te: spesso basta chiedere "inizializza una repo e collegala a GitHub".

Committare e pushare — i due gesti fondamentali

Sono le due parole che sentirai di più. Sono due passi distinti:

✏️
Working directory
i file che stai modificando
 
→
📸
Commit
istantanea, sul tuo PC
git commit -m "…"
→
☁️
Push
online, su GitHub
git push

Premi il pulsante per seguire una modifica lungo i tre stati.

Committare

git add sceglie quali modifiche includere, poi git commit -m "messaggio" le sigilla in un'istantanea.

Il messaggio spiega cosa hai cambiato ("aggiunta pagina di login"). Resta tutto sul tuo computer.

Pushare

git push prende i commit fatti in locale e li manda su GitHub.

Finché non pushi, il tuo lavoro è solo sul tuo PC. Dopo il push è al sicuro nel cloud e visibile agli altri.

02 I comandi essenziali

Guidare la sessione con i comandi /

Dentro Claude Code, digitando / apri i comandi rapidi. Questi sono quelli che userai davvero ogni giorno.

/clear

Azzera il contesto. Come iniziare una conversazione nuova: Claude dimentica tutto quello detto finora.

/compact

Riassume e comprime il contesto mantenendo il filo del discorso. Continui lo stesso lavoro con meno "peso".

/model

Cambia modello. Passi tra Opus, Sonnet e Haiku a seconda che ti serva più profondità o più velocità.

/effort

Cambia il livello di ragionamento della sessione (da low a max): quanto a fondo Claude pensa prima di agire.

/init

Genera il CLAUDE.md del progetto: un file con le istruzioni e il contesto che Claude leggerà a ogni sessione.

/agents

Crea e gestisce i subagent: aiutanti specializzati a cui Claude delega pezzi di lavoro in parallelo.

/mcp

Stato e gestione dei server MCP: i connettori che collegano Claude a strumenti esterni (Gmail, GitHub, database…).

/usage

Mostra l'utilizzo: token e costo stimato della sessione e, sui piani Pro/Max, quanto hai consumato rispetto ai limiti.

/resume

Riprende una sessione precedente. Ritrovi una conversazione di ieri esattamente dov'era rimasta, con tutto il suo contesto.

03 Gestire il contesto

/clear o /compact?

Il "contesto" è tutto ciò che Claude ha in memoria nella sessione. Più si riempie, più diventa lento e dispersivo. Questi due comandi lo gestiscono in modi opposti.

/clear — riparti da zero

Usalo quando cambi argomento: hai finito un compito e ne inizi un altro scollegato.

Cancella tutto. Nessun residuo del lavoro precedente a confondere le acque.

/compact — alleggerisci, ma resta

Usalo quando stai ancora sullo stesso lavoro ma la conversazione è diventata lunga.

Claude riassume quanto fatto e prosegue: mantiene il filo, libera spazio.

Auto-compact. Quando il contesto si riempie davvero, Claude Code fa il /compact da solo per non bloccarsi. Ma farlo tu al momento giusto — a fine di un sotto-compito — tiene la sessione più pulita.
04 Le modalità operative

Quanta libertà dai a Claude — Shift+Tab

Premendo Shift+Tab cicli tra le modalità: cambiano quanto Claude chiede il permesso prima di agire. Provale qui sotto.

● NORMAL
Normal / Edit

Chiede conferma prima di ogni azione. Il default prudente: vedi e approvi ogni modifica.

Auto-accept edits

Applica le modifiche ai file senza chiedere. Veloce quando ti fidi della direzione.

Plan mode

Esplora e progetta senza toccare nulla, poi ti propone un piano da approvare prima di agire.

Auto mode

Claude procede in autonomia fino al risultato, incatenando le azioni senza fermarsi a ogni passo.

Regola pratica: Plan mode per compiti nuovi o delicati (prima capisci, poi agisci) · Normal per lavorare con controllo · Auto-accept / Auto quando la strada è chiara e vuoi andare spedito.

05 Effort & ragionamento

Quanto far ragionare Claude

Puoi decidere quanto a fondo Claude ragiona prima di rispondere. Il comando /effort imposta il livello su una scala di cinque: più sali, più pensa — ma impiega più tempo. Trascina il cursore.

low
ragionamento minimo
low medium high xhigh max

Due leve vicine. /effort dice quanto ragionare; /model sceglie con quale modello (Opus ragiona più a fondo, Haiku è più rapido). I modelli recenti dosano già il ragionamento in automatico in base alla difficoltà — con /effort alzi o abbassi il tetto. E Plan mode è il posto naturale dove tenerlo alto: stai progettando, non ancora eseguendo.

Quando alzarlo: bug ostici, decisioni di architettura, problemi che richiedono più passaggi logici. Per compiti semplici, tenerlo basso è più veloce ed economico.

06 Scorciatoie utili

Tre gesti da tastiera che salvano tempo

Esc

Interrompi Claude. Sta andando nella direzione sbagliata? Fermalo subito e riscrivi il prompt.

Esc Esc oppure /rewind

Torna indietro. Riavvolge la conversazione a un punto precedente, cancellando tutto ciò che è venuto dopo.

Ctrl+V

Incolla un'immagine nel prompt: uno screenshot di un errore, un mockup, un grafico. Su macOS usa Ctrl+V (con ⌘V incolli solo il percorso del file). In alternativa, trascina il file nel terminale.

Windows o macOS? Queste tre scorciatoie sono identiche sulle due piattaforme. In generale, le combinazioni che su macOS usano ⌘ Cmd diventano Ctrl su Windows — ma Esc resta Esc ovunque.
07 Lavorare dal telefono

Continua la sessione da mobile — /rc

Hai avviato Claude Code sul computer e devi allontanarti? Con Remote Control prosegui e monitori la stessa sessione dal telefono, dal tablet o da un browser qualsiasi.

Come funziona
1

Nel terminale digiti /remote-control (in breve /rc). Claude mostra un link e un QR code.

2

Apri l'app Claude sul telefono (iOS/Android) nella scheda Code, oppure inquadri il QR, oppure vai su claude.ai/code da browser.

3

Scrivi e segui l'avanzamento dal telefono: la conversazione resta sincronizzata con il terminale sul computer.

Attenzione: la sessione gira sul tuo computer, che deve restare acceso. Se chiudi il terminale o lo spegni, la sessione remota finisce. Per far proseguire il lavoro a computer spento serve il cloud → sezione successiva.
08 Lavorare in cloud

Va avanti anche a computer spento

Claude Code on the web esegue i task su macchine in cloud gestite da Anthropic, non sul tuo PC. Puoi chiudere il browser, spegnere il computer, andare offline: la sessione continua e la ritrovi dov'era.

Si avvia da claude.ai/code o da terminale con claude --cloud "sistema il bug X". Collega GitHub una volta (tramite l'app GitHub di Claude): da lì la macchina cloud clona il progetto, lavora su un branch, apre una Pull Request e — con l'auto-fix attivo — reagisce agli errori dei test (CI) e ai commenti di revisione.

Remote Control o cloud? Non confonderli

Remote Control (/rc)

La sessione gira sul tuo computer, che deve restare acceso.

Tu la controlli da telefono/browser. Ideale per seguire un lavoro già avviato mentre sei in giro.

Claude Code on the web

La sessione gira sui server di Anthropic: va avanti anche a computer spento.

Dialoga con GitHub (branch + Pull Request). Ideale per delegare un task e ritrovarlo pronto.

Disponibile sui piani Pro, Max, Team ed Enterprise (in fase di preview).
